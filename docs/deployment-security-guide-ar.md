# دليل النشر والحماية

## 1. هل يمكن تشغيل النظام أونلاين؟
نعم. يمكن تشغيله:
- على جهاز محلي داخل الفرع.
- على سيرفر داخلي داخل الشركة.
- على VPS أو Windows Server أو Linux Server مع نشر Web.

## 2. الشكل الأنسب للنشر
- `نسخة محلية`: مناسبة لفرع واحد أو تجربة أولية.
- `نسخة مركزية أونلاين`: مناسبة لسلاسل الفروع مع وصول موحد.

## 3. متطلبات النشر الأونلاين
- دومين أو Subdomain.
- شهادة SSL.
- Reverse Proxy مثل Nginx أو IIS.
- نسخ احتياطي مجدول.
- جدار حماية.
- حصر الوصول على المنافذ المطلوبة فقط.

## 4. حماية النظام من الاختراق
- لا تفتح منفذ قاعدة البيانات للإنترنت.
- شغل النظام خلف Proxy مع HTTPS فقط.
- استخدم كلمات مرور قوية وغير الافتراضية.
- غيّر اسم مستخدم المدير أو أضف مديرًا جديدًا ثم عطّل الافتراضي.
- فعّل نسخًا احتياطية يومية واختبر الاستعادة.
- لا تسمح بالوصول الإداري إلا من عناوين أو أجهزة موثوقة.
- راقب سجل المراجعة باستمرار.
- حدث Node.js والنظام التشغيلي دوريًا.

## 5. توصيات قبل البيع أو التسليم
- سلّم العميل نسخة `Clean` فارغة.
- احتفظ أنت بنسخة `Demo` للعرض فقط.
- لا تسلم أي حركات فعلية تخص عميل آخر.
- أنشئ كلمة مرور خاصة لكل عميل.
- فعّل الثيم المناسب واسم النشاط قبل التسليم.

## 6. ما الذي يلزم لو أردت بيع النظام كخدمة SaaS؟
- فصل قاعدة بيانات كل عميل.
- إضافة نظام ترخيص أو اشتراك.
- إضافة دومين فرعي لكل عميل أو بيئة مستقلة.
- إضافة مراقبة Logging وHTTPS وRate Limiting.
- إضافة سياسة تحديث وترحيل بيانات محسوبة.
